内容日历不是把标题排进表格那么简单。真正有用的内容日历需要回答三个问题：为什么写这篇、写给谁、写完后如何判断有没有价值。很多内容站被做成低质量资源库，问题不在于没有更新，而在于更新计划只追求数量，没有围绕用户任务、搜索意图和站内内容结构形成积累。

这个工作流适合用 AI 辅助制定 4 周到 8 周的内容计划。它不会替你判断所有选题是否值得做，但可以帮助你把关键词、栏目、文章类型、内链和复盘指标放到同一张表里，避免“今天想到什么就写什么”。

## 这个工作流解决什么问题

内容站持续更新最难的是稳定选题和排期。内容日历工作流可以把“想写什么”变成“什么时候写、写给谁、为什么写”。

它尤其适合以下场景：

- 你已经有一个内容站，但更新节奏不稳定
- 你有很多关键词，却不知道先写哪一批
- 首页、专题页、文章页之间缺少内链规划
- 想为 AdSense、SEO 或自然流量做长期内容积累
- 团队多人参与写作，需要统一选题标准和交付节奏

它不适合以下情况：

- 只想一次生成上百个标题然后批量发布
- 没有明确目标用户，只想追热点
- 每篇内容都只有工具链接或提示词代码块，没有原创解释
- 没有人负责复盘 Search Console、点击率和收录情况

## 输入信息

使用前建议准备这些信息：

- 内容主题
- 目标用户
- 核心栏目
- 目标关键词
- 更新频率
- 商业化目标
- 已有内容列表
- 当前重点专题
- 不想覆盖或暂时不适合写的主题

如果你已经有 Google Search Console 数据，可以补充：

- 曝光高但点击低的查询
- 已有排名但内容较薄的页面
- 有点击但缺少后续内链的页面
- 尚未收录或表现不稳定的页面

## 工作流步骤

### 第 1 步：定义内容目标

先让 AI 根据目标用户和网站定位，拆出内容目标。不要直接让它生成标题。

```text
你是一名内容策略编辑。请根据以下网站定位和目标用户，拆解未来 4 周内容更新目标。

请输出：
1. 核心用户任务
2. 用户搜索意图
3. 应优先建设的内容簇
4. 不建议优先写的内容
5. 每类内容的判断标准

网站定位：
[填写网站定位]

目标用户：
[填写目标用户]

已有栏目：
[填写栏目]
```

这一步的目标是避免 AI 一上来生成一堆看起来热闹但没有结构的标题。

### 第 2 步：生成选题池

在目标清楚后，再让 AI 生成选题池。

```text
请基于上一步内容目标，生成 30 个候选选题。

每个选题请包含：
- 标题
- 对应栏目
- 目标关键词
- 搜索意图
- 适合内容类型：Prompt / Guide / Workflow / Tool / Agent / Skill
- 适合读者
- 需要补充的原创经验或案例
- 低价值风险
```

这里特别要保留“需要补充的原创经验或案例”和“低价值风险”两列。它们可以提醒你：有些标题如果只是复制工具介绍，很容易变成 Thin Content。

### 第 3 步：按搜索意图和内容簇分类

选题池生成后，不要马上排期。先按内容簇分类：

- 入门指南
- 可复制模板
- 完整工作流
- 工具教程
- 对比评测
- 避坑和复盘
- 专题支撑文章

如果某个内容簇只有 1 篇文章，先不要急着做专题入口。专题页最好至少有 4-6 篇相关内容支撑，否则容易看起来像空壳导航。

### 第 4 步：设定发布时间

建议先做 4 周计划，而不是全年计划。原因很现实：AI 工具变化快，搜索表现也需要观察。一次排太久，后面很可能全部重排。

输出示例：

| 日期 | 标题 | 栏目 | 关键词 | 目标 | 原创补充点 |
|---|---|---|---|---|---|
| 第 1 周 | AI 写周报 Prompt | Prompt | AI 周报 Prompt | 获取长尾流量 | 加入失败输出和修改示例 |
| 第 2 周 | Claude Code 入门 | Tool | Claude Code 教程 | 工具教程入口 | 加入真实项目使用流程 |
| 第 3 周 | 用 AI 写 PRD | Workflow | AI PRD | 产品经理人群 | 加入需求澄清清单 |

### 第 5 步：规划内链

内容日历不能只看发布日期，还要规划内链。每篇文章至少考虑：

- 从哪个专题页链接进来
- 链向哪 2-3 篇相关内容
- 是否支撑某个栏目页或任务页
- 是否需要在首页或资源页露出

可以让 AI 输出：

```text
请为这 12 篇内容规划站内链接。

每篇输出：
1. 推荐上游入口
2. 推荐下游阅读
3. 应链接的专题页
4. 应避免重复曝光的首页模块
```

### 第 6 步：定义复盘指标

内容发布后不要只看是否上线。至少跟踪：

- 是否被 Google 收录
- Search Console 曝光和点击
- 页面停留和跳出
- 站内搜索是否出现相关关键词
- 是否有用户反馈或投稿
- 是否需要补充案例和截图

## 常见失败案例

### 失败 1：AI 生成的标题太像批量模板

例如：

```text
10 个提高效率的 AI 工具
AI 工具大全
最好用的 AI 工具推荐
```

这些标题很容易变成泛泛而谈的工具汇编。更好的做法是让标题围绕具体任务：

```text
用 AI 把会议记录整理成行动项：从输入格式到责任人追踪
```

### 失败 2：内容日历只排标题，不排工作量

有些文章只是 800 字指南，有些文章需要截图、测试、对比和案例。如果日历不标记工作量，排期会失真。

建议增加字段：

- 预计工作量：低 / 中 / 高
- 是否需要截图
- 是否需要真实案例
- 是否需要更新旧文

### 失败 3：忽略旧文翻新

AdSense 和 SEO 都不只看新文章数量。旧文如果过薄、过时或没有原创经验，也会拖累整体质量。内容日历里应保留“旧文重写”任务，例如：

- 扩写核心 Prompt 页面
- 给工具教程补充真实配置步骤
- 给 Workflow 页面补充失败案例
- 合并只有几段文字的碎片页

## 可复制的内容日历 Prompt

```text
你是一名内容策略编辑和 SEO 内容运营。请帮我为一个 AI 生产力内容站制定未来 4 周内容日历。

网站定位：
[填写定位]

目标用户：
[填写用户]

已有栏目：
[Prompt / Skill / Agent / Tool / Workflow / Guide]

已有核心内容：
[粘贴已有文章标题和 URL]

目标：
[例如：提升自然搜索流量、准备 AdSense 审核、完善 AI 写作专题]

请输出：
1. 内容策略判断
2. 未来 4 周内容日历表
3. 每篇文章的搜索意图
4. 每篇文章必须补充的原创经验或案例
5. 推荐内链
6. 低价值内容风险提醒
7. 发布后复盘指标

要求：
- 不要只生成标题
- 不要建议批量生产低质量工具汇编
- 优先选择能形成专题和内链的内容
- 标记哪些旧文需要翻新
```

## 使用建议

不要让 AI 一次生成全年计划。先做 4 周内容日历，每周根据 Search Console 数据调整。对于新站或准备 AdSense 审核的网站，我更建议把一部分时间用于“翻新旧文”，而不是只追求新文章数量。

一个更稳妥的节奏是：每周 2 篇新内容 + 2 篇旧文深度翻新。新内容负责扩展覆盖面，旧文翻新负责提高整体质量和可信度。
