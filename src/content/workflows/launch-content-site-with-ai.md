用 AI 搭建内容站并不难，难的是把“能上线”推进到“值得长期维护”。很多站点一开始只生成几十篇短文、套一个模板、部署到 Cloudflare Pages，然后就等待收录和广告审核。这样做通常会遇到三个问题：定位不清、内容太薄、技术细节没有检查。

这个工作流把内容站上线拆成 6 个阶段：定位、栏目、内容种子、技术实现、部署检查和上线后复盘。AI 可以帮助你生成信息架构、选题池、页面代码和检查清单，但最终仍需要人工判断内容是否有独特价值、页面是否可用、合规信息是否完整。

## 这个工作流解决什么问题

很多人想做内容站，但会卡在定位、技术选型、内容规划和部署细节上。AI 可以帮助把这些步骤拆开，并生成计划、代码和检查清单。

适合：

- 想用 Astro、Markdown 和 Cloudflare Pages 做静态内容站
- 已经有一个主题方向，但不知道怎么拆栏目
- 希望用 AI 辅助生成首批选题和页面结构
- 需要上线前检查 SEO、sitemap、隐私政策、About 页面
- 想为 AdSense 或长期 SEO 做内容质量准备

不适合：

- 想自动生成大量低质量文章并直接发布
- 没有维护计划，只想短期堆页面
- 内容涉及医疗、法律、投资等高风险领域，但没有专业资质
- 期待 AI 一次性完成从定位到商业化的所有决策

## 工作流总览

1. 明确站点定位和目标读者。
2. 设计栏目结构和 URL 结构。
3. 生成首批选题，并按搜索意图筛选。
4. 初始化 Astro 内容站，建立内容集合和模板。
5. 部署到 Cloudflare Pages，并检查构建、域名、缓存。
6. 配置 Search Console、sitemap、robots、基础分析。
7. 准备 AdSense 或其他商业化审核。
8. 上线后每周复盘内容质量和索引情况。

## Step 1：定位和栏目

不要一开始就让 AI 写代码。先让它帮你澄清站点定位：

```text
我想搭建一个内容站，请帮我设计定位和栏目结构。

输入信息：
- 主题方向：
- 目标读者：
- 读者常见问题：
- 我能提供的真实经验或资源：
- 不想覆盖的内容：
- 未来可能的商业化方式：

输出要求：
1. 网站一句话定位
2. 目标读者画像
3. 3-6 个一级栏目
4. 每个栏目的内容边界
5. 不建议做的内容方向
6. 第一批可写主题
```

示例定位：

```text
网站定位：面向中文知识工作者的 AI 生产力内容站，提供 Prompt、Workflow、Tool 和 Agent 的实用教程。

栏目：
- Prompts：可直接复制使用的提示词
- Workflows：完成具体工作的步骤
- Tools：AI 工具教程和选型
- Agents：面向特定角色的智能体设计
- Guides：更长的入门和选型指南
```

好的定位应该能帮你排除内容。如果一个主题无法判断该放哪个栏目，说明栏目边界可能还不清楚。

## Step 2：生成内容种子

让 AI 批量生成选题后，不要全部发布。建议先按搜索意图和内容能力筛选。

```text
请基于以下网站定位生成 50 个内容选题。

要求：
- 每个选题包含：标题、目标读者、搜索意图、需要的真实案例、内容难度、是否适合新站优先写
- 区分信息型、教程型、对比型、模板型
- 标记容易写成低价值内容的选题
- 不要只生成泛泛标题

网站定位：
[填写]

栏目结构：
[填写]
```

筛选标准：

| 标准 | 问题 |
|---|---|
| 搜索意图 | 读者搜索这个词时到底想解决什么问题？ |
| 独特性 | 我是否能提供真实经验、示例、截图或判断？ |
| 长期价值 | 半年后这篇内容是否仍然有用？ |
| 可维护性 | 工具变化后是否容易更新？ |
| 风险 | 是否容易涉及专业建议或夸大承诺？ |

对于新站，优先写“能讲清楚具体步骤”的教程，而不是泛泛的趋势观点。

## Step 3：开发 Astro 内容站

如果选择 Astro，可以让 AI 先生成项目结构，而不是一次生成所有代码：

```text
请帮我设计一个 Astro 内容站的目录结构。

要求：
- 使用 Markdown 或 MDX 管理内容
- 支持文章列表页、详情页、分类页和专题页
- 支持 title、description、pubDate、updatedDate、tags、draft 等 frontmatter
- 支持 sitemap、robots、基础 SEO meta
- 给出每个目录的用途

站点栏目：
[填写]
```

一个常见结构：

```text
src/
  content/
    prompts/
    workflows/
    tools/
    guides/
  layouts/
  pages/
    topics/
    about.astro
    privacy-policy.astro
    terms.astro
```

AI 生成代码后，建议逐文件检查，不要直接复制一大段未知代码上线。尤其要检查路由、内容集合 schema、日期格式、draft 过滤和 sitemap 输出。

## Step 4：部署到 Cloudflare Pages

部署前准备：

- GitHub 仓库能正常构建
- `npm run build` 本地通过
- Cloudflare Pages 构建命令和输出目录正确
- 自定义域名已配置 DNS
- 生产环境没有暴露测试内容
- `draft: true` 页面不会出现在列表和 sitemap 中

常见 Cloudflare Pages 配置：

```text
Build command: npm run build
Build output directory: dist
Node version: 与项目要求一致
```

如果构建失败，可以让 AI 分析日志：

```text
下面是 Cloudflare Pages 构建失败日志。请帮我判断失败原因，并给出最小修改建议。

要求：
- 先定位错误行
- 区分依赖问题、代码问题、环境变量问题和路径问题
- 不要建议大规模重构

日志：
[粘贴日志]
```

## Step 5：上线前 SEO 和合规检查

内容站上线前至少检查：

- 首页是否说明网站提供什么价值
- 每个核心栏目是否有可读的说明，而不是只有列表
- 文章页是否有标题、描述、发布时间、更新时间
- 是否生成 sitemap
- robots.txt 是否允许抓取公开页面
- 是否有 About、Privacy Policy、Terms 页面
- 是否有清晰联系方式或站点维护者说明
- 是否没有明显未来日期、空页面、重复模板内容
- 404 页面是否可用

如果准备 AdSense，还要额外关注：

- 不要只发布短内容或纯 Prompt 列表
- 每篇核心内容最好包含使用场景、步骤、示例、失败案例和人工检查清单
- 隐私政策中说明 cookies、广告和分析工具
- 页面不要在审核前放置过多占位广告位
- 确保导航能访问主要内容，不要依赖搜索框才能发现页面

## 使用案例：AI 生产力内容站

### 场景

一个站点围绕 Prompt、Workflow、Tool、Agent 和 Guide 做中文 AI 生产力教程，希望用 Astro + Cloudflare Pages 上线，并逐步准备搜索收录和广告审核。

### 推荐执行顺序

1. 先完成首页、专题页、About、隐私政策和条款。
2. 每个栏目至少准备几篇深度内容，而不是只放标题和短描述。
3. 先写高意图内容，例如周报 Prompt、PRD Prompt、内容日历 Workflow。
4. 给主题页增加导读和阅读路径，降低“薄列表页”风险。
5. 上线后检查 Search Console 收录和页面体验。
6. 再根据数据翻新低深度页面。

### 不建议的做法

```text
一次性让 AI 生成 100 篇短文章，直接发布，等待 AdSense 审核。
```

这类站点很容易被识别为低价值内容。更稳妥的是少量高质量页面先上线，再持续扩展。

## 常见失败案例

### 失败 1：栏目太多，内容太少

新站常见问题是导航里有 10 个栏目，但每个栏目只有 1-2 篇短文。用户和搜索引擎都会觉得站点还没准备好。

建议：先保留 3-5 个核心栏目，把每个栏目做扎实。

### 失败 2：只重视代码，不重视内容质量

Astro、Cloudflare、sitemap 都配置好了，但文章只有几百字，缺少案例和人工判断。这时技术上已经上线，内容上仍然不够成熟。

建议：用内容翻新清单持续补充场景、示例、失败输出、检查清单和内部链接。

### 失败 3：上线后不复盘

内容站不是一次性项目。上线后至少每周检查：哪些页面被收录，哪些页面有曝光，哪些页面跳出后没有继续阅读。

## 可复制完整 Prompt

```text
你是一名内容站产品和技术顾问。请帮我制定一个从 0 到上线的内容站计划。

我的信息：
- 主题方向：
- 目标读者：
- 我能持续产出的内容类型：
- 技术栈倾向：Astro / Next.js / WordPress / 其他
- 部署方式：Cloudflare Pages / Vercel / 其他
- 商业化目标：AdSense / 联盟营销 / 产品线索 / 暂无

请输出：
1. 网站定位
2. 栏目结构
3. 首批 30 个选题，并标记优先级
4. 页面模板清单
5. 技术实现步骤
6. 部署检查清单
7. SEO 和合规检查清单
8. 上线后 4 周复盘计划
9. 低价值内容风险和避免方式
```

## 上线前检查清单

- 首页是否清楚说明网站主题和适合谁？
- 栏目是否有明确边界？
- 首批内容是否有真实案例或操作步骤？
- 是否有 About、Privacy Policy、Terms？
- sitemap、robots、canonical 是否正常？
- 本地和生产构建是否通过？
- 站点是否没有未来日期和草稿泄露？
- 404、移动端、内部链接是否可用？
- 是否配置 Search Console 并提交 sitemap？
- 是否有未来 4 周内容更新计划？

## 结论

AI 可以让内容站从想法到上线更快，但不能跳过定位、内容质量和上线后复盘。真正可持续的内容站不是“生成很多页面”，而是围绕明确读者持续解决问题。先把少量核心页面做深，再扩展栏目和规模，通常比一次性铺大量薄内容更稳。
