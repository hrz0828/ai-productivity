运营策划最怕两件事：方案看起来热闹，但没有资源支撑；执行清单很长，但和目标指标没有关系。AI 运营策划 Agent 的价值不是凭空生成“创意活动”，而是帮助运营人员从目标、用户、资源和渠道出发，把活动拆成可执行、可复盘的计划。

这个 Agent 适合活动运营、内容运营、增长运营和独立项目负责人使用。它可以辅助生成活动定位、内容排期、渠道节奏、物料清单、风险预案和复盘指标，但预算、渠道权限、品牌边界和最终承诺必须由人工确认。

## Agent 角色

AI 运营策划 Agent 帮助运营人员从目标出发，设计活动方案、内容计划、渠道节奏和复盘指标。

它应该像一个谨慎的运营搭档：先问清楚目标和资源，再给方案；先判断可行性，再扩展创意；先定义指标，再安排动作。

## 适合场景

适合：

- 新产品上线活动策划
- 内容专题或栏目运营
- 社群活动和用户召回
- 内容站专题推广
- 小红书、公众号、邮件、站内资源联动
- 活动结束后的复盘框架整理

不适合：

- 预算、渠道、权益都不明确，却要求完整落地方案
- 需要法律、财务、合同审批的活动，但没有人工复核
- 用虚假优惠、夸大收益或误导性文案拉转化
- 让 AI 承诺用户权益、价格、奖励或合作条款

## 输入要求

使用前建议准备：

```text
活动目标：
目标用户：
核心产品或内容：
预算和资源：
时间周期：
可用渠道：站内 / 公众号 / 小红书 / 邮件 / 社群 / 广告
已有素材：文章 / 海报 / 案例 / 用户反馈 / 数据
预期指标：曝光 / 点击 / 注册 / 留资 / 购买 / 复访
限制条件：不能承诺什么、不能使用什么素材
```

如果目标只写“提升影响力”，Agent 应先追问：提升谁的影响力？通过什么渠道？用什么指标判断？

## 系统提示词

```text
你是一名运营策划 Agent，擅长活动策划、内容排期和数据复盘。

工作原则：
1. 先确认活动目标、目标用户、资源、渠道和时间周期。
2. 不编造预算、权益、用户数据或渠道能力。
3. 活动玩法必须能落到具体执行清单。
4. 每个动作都要对应目标指标或用户路径。
5. 对高风险承诺、夸大宣传和合规风险进行提醒。
6. 如果信息不足，先提出澄清问题，不要直接生成完整方案。

请根据用户目标输出：
1. 活动定位
2. 用户洞察
3. 核心玩法
4. 内容计划
5. 渠道节奏
6. 执行清单
7. 数据指标
8. 风险和备选方案
9. 复盘方式
```

## 工作流

1. **澄清目标**：活动是为了曝光、转化、留存，还是内容资产沉淀？
2. **拆用户路径**：用户从哪里看到、为什么点击、进入后做什么。
3. **设计玩法**：根据资源选择专题、挑战、模板领取、直播、社群打卡等形式。
4. **排内容节奏**：预热、上线、提醒、收尾、复盘。
5. **列执行清单**：物料、页面、文案、渠道、负责人、时间点。
6. **定义指标**：曝光、点击、收藏、注册、转化、复访等。
7. **做风险预案**：资源不足、审核延迟、转化低、用户反馈异常。

## 使用案例：AI 周报模板专题活动

### 场景

你要为 AI 内容站做一次“AI 周报模板专题”活动，目标是让用户进入专题页，阅读周报 Prompt，并继续浏览相关职场写作内容。

### 输入给 Agent

```text
活动目标：提升 AI 周报 Prompt 页面和相关职场写作页面的访问与继续阅读。
目标用户：需要写周报的职场新人、项目成员、运营和产品人员。
资源：已有周报 Prompt、邮件 Prompt、OKR Prompt、公众号文章工作流。
渠道：站内首页、AI 写作专题页、小红书、公众号。
周期：7 天。
限制：不夸大效率提升，不承诺“领导一定满意”。
```

### Agent 输出示例

```markdown
## 活动定位
用 7 天时间把“周报写作”做成一个职场 AI 写作专题，帮助用户从流水账改成结构化汇报。

## 内容计划
- Day 1：站内专题入口上线，推荐周报 Prompt。
- Day 2：发布小红书笔记《周报写成流水账？先改这 3 点》。
- Day 3：公众号文章讲解周报输入示例和输出示例。
- Day 5：补充邮件和 OKR 相关内链，形成职场写作路径。
- Day 7：复盘专题页点击、相关文章浏览和搜索表现。

## 指标
- 专题入口点击
- 周报 Prompt 页面访问
- 相关页面继续阅读点击
- 小红书收藏和评论问题
```

## 常见失败输出

### 失败 1：玩法很大，但资源不够

```text
联合 20 位行业专家做线上峰会，并投放全平台广告。
```

如果没有嘉宾、预算和投放能力，这不是方案。可以要求 Agent 改成低资源版本：

```text
请基于现有文章和社媒账号，生成一个不需要外部嘉宾和广告预算的活动方案。
```

### 失败 2：指标和动作无关

活动动作是“发 10 篇内容”，指标却写“收入增长 300%”。如果中间没有注册、咨询、购买路径，这个指标不可用。

### 失败 3：文案过度承诺

```text
领取这个模板，让你的工作效率翻 10 倍。
```

更稳妥：

```text
领取这个周报模板，把零散事项整理成更清晰的汇报结构。
```

## 变体 Prompt：生成执行清单

```text
请把下面的运营活动方案拆成执行清单。

要求：
- 按时间顺序排列
- 每项包含事项、负责人、截止时间、所需素材、依赖和验收标准
- 负责人未知时写“待确认”
- 标记可能阻塞活动上线的事项

活动方案：
[粘贴]
```

## 变体 Prompt：活动复盘

```text
请根据下面的活动数据生成复盘报告。

要求：
- 对比目标和实际结果
- 区分渠道表现、内容表现和转化路径问题
- 不把相关性当成因果
- 输出后续优化动作

活动目标：
[填写]

数据：
[曝光、点击、访问、注册、收藏、评论、转化等]
```

## 人工检查清单

- 活动目标是否明确？
- 目标用户是否具体？
- 活动玩法是否符合现有资源？
- 每个渠道是否有对应内容和负责人？
- 指标是否能被真实追踪？
- 是否存在夸大承诺、虚假优惠或合规风险？
- 执行清单是否有时间点和依赖？
- 是否准备了低成本备选方案？
- 复盘时是否能判断哪一步出了问题？

## 注意事项

活动方案必须结合真实资源。AI 可以发散创意，但预算、渠道能力、品牌边界、用户权益和执行周期需要人工确认。一个好的运营策划 Agent 不应该只给“热闹”的创意，而应该帮助你把目标、路径、动作和指标对齐。
