AI PPT 策划 Agent 的重点不是“自动生成漂亮页面”，而是先把演示目标、受众关心点和叙事结构讲清楚。很多 PPT 失败并不是因为模板不好看，而是页面顺序不符合听众问题：该先讲结论时讲了背景，该给证据时堆了口号，该提出决策时只做了总结。

这个 Agent 适合在进入 PowerPoint、Keynote、Canva 或 Gamma 之前使用。它可以帮助你从主题生成演示主线、页面大纲、每页核心观点、图示建议和演讲备注，但真实数据、截图、视觉排版和现场表达仍需要人工完成。

## Agent 角色

AI PPT 策划 Agent 专注于演示内容策划，而不是单纯生成页面文字。它会先理解受众和目标，再设计叙事结构。

它应该像一个演示顾问：先问“这场演示要让谁做什么决定”，再决定页面顺序和信息密度。

## 目标

- 澄清演示目的
- 判断受众关心点
- 设计叙事主线
- 生成页面级大纲
- 提供图示和表达建议
- 标记需要补充的数据、截图和案例
- 帮助准备听众追问

## 适合场景

适合：

- 工作汇报
- 产品方案
- 项目复盘
- 融资路演
- 培训分享
- 内容站或产品介绍
- 对外合作提案

不适合：

- 没有事实材料，只想让 AI 凑页数
- 涉及融资、财务、法律承诺但没有专业复核
- 需要严格品牌视觉规范，却完全依赖 AI 排版
- 只追求“高级感”，不明确听众和目标

## 输入要求

建议每次给 Agent 提供：

```text
演示主题：
目标受众：
演示目标：告知 / 说服 / 决策 / 培训 / 复盘
演示时长：
页数范围：
已有材料：数据 / 截图 / 文档 / 案例
必须出现的信息：
不能提及的信息：
希望听众最后采取的行动：
```

如果这些信息缺失，Agent 应先提出澄清问题，而不是直接生成 10 页 PPT。

## 系统提示词

```text
你是一名资深 PPT 策划 Agent。请帮助用户把主题转化为清晰、有说服力的演示结构。

工作方式：
1. 先确认受众、目标、时长、页数和决策场景。
2. 给出 2-3 种叙事结构选择，并说明适合的场景。
3. 生成页面级大纲。
4. 每页输出核心观点、要点、图示建议、素材需求和演讲备注。
5. 标记需要用户补充的数据、截图或案例。
6. 不编造业务数据、用户反馈、收入预测或合作承诺。
7. 如果内容过多，主动建议合并和删减页面。
```

## 工作流

1. **澄清目标**：这份 PPT 是同步信息、争取资源，还是推动决策？
2. **分析受众**：听众最关心结果、风险、成本、收益还是执行路径？
3. **选择结构**：结论先行、问题-方案、过去-现在-未来、用户旅程等。
4. **生成页面大纲**：每页只承载一个核心结论。
5. **补素材需求**：数据、截图、流程图、案例、对比表。
6. **生成备注**：帮助演讲者讲清每页逻辑，而不是照读文字。
7. **准备追问**：预测听众可能质疑的点。

## 使用案例：策划一次项目复盘汇报

### 场景

你需要向团队汇报 AI 内容站第一阶段建设结果，包括文章数量、功能进展、SEO 准备和 AdSense 准备。

### 示例任务

```text
帮我策划一份 10 页 PPT，用于汇报 ai-productivity 第一阶段建设成果。

受众：项目参与者和负责人。
目标：总结成果、暴露问题、明确下一步计划。
时长：12 分钟。
已有材料：内容页面清单、构建结果、AdSense 整改计划、专题页更新。
限制：不写未经验证的流量和收入预期。
```

### Agent 输出示例

```markdown
## 推荐主线：过去-现在-下一步
适合复盘汇报，先说明已完成事项，再暴露风险，最后提出下一阶段计划。

### 第 1 页：本阶段目标是提升内容质量和审核准备度
- 核心观点：本阶段不是单纯增加页面，而是降低低价值内容风险。
- 图示建议：目标拆解卡片。
- 素材需求：整改计划摘要。

### 第 2 页：已优先完成信任页面和高意图内容页
- 核心观点：About、隐私政策、条款和核心 Prompt 页面已完成扩写。
- 图示建议：完成清单。
- 素材需求：页面列表和更新时间。

### 第 3 页：内容扩写标准从“模板”升级为“教程”
- 核心观点：每页补充场景、示例、失败案例和人工检查清单。
- 图示建议：前后对比。
```

### 人工调整

如果听众更关注业务结果，应增加流量、收录、内容质量和商业化准备；如果听众更关注开发，应增加架构和部署部分。

## 常见失败输出

### 失败 1：页面标题都是名词

```text
项目背景
内容建设
后续计划
```

更好的标题应该表达结论：

```text
第一阶段重点从“页面数量”转向“内容可信度”
```

### 失败 2：每页要点太多

一页放 8 个 bullet 会让听众抓不住重点。可以要求 Agent 压缩为：一个结论、三个要点、一种图示。

### 失败 3：缺少素材提醒

如果 Agent 只写页面文字，不提示需要截图、数据和案例，后续制作会变成空话。系统提示词中要明确“标记素材需求”。

### 失败 4：编造业务结果

```text
本阶段带来显著流量增长。
```

如果没有数据，应该写：

```text
流量和收录表现需要等待 Search Console 数据进一步验证。
```

## 变体 Prompt：生成听众追问

```text
请站在目标受众角度，预测这份 PPT 可能被追问的问题。

目标受众：
[填写]

页面大纲：
[粘贴]

请输出：
1. 可能被问的 10 个问题
2. 每个问题背后的真实担心
3. 需要准备的数据或证据
4. 回答时不应过度承诺的边界
```

## 变体 Prompt：压缩页面

```text
请把这份 PPT 大纲压缩到 [页数] 页。

要求：
- 保留演示目标必须的信息
- 合并重复页面
- 删除没有证据支撑的内容
- 每页保留一个核心结论
- 标记需要补充素材的位置
```

## 人工检查清单

- 演示目标是否清楚？
- 受众关心的问题是否提前回答？
- 叙事结构是否适合当前场景？
- 每页是否只有一个核心观点？
- 是否有足够数据、截图或案例支撑？
- 是否存在未经验证的承诺或预测？
- 页数是否匹配演示时长？
- 结尾是否明确下一步行动？
- 是否准备了听众追问的回答？

## 局限性

Agent 能帮助建立结构，但视觉设计和素材质量仍需要人工把控。最终 PPT 的说服力来自真实证据、清晰取舍和演讲者对听众问题的理解，而不只是页面数量和模板效果。
