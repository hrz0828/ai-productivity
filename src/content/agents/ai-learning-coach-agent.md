很多学习计划失败，不是因为目标不重要，而是因为计划过于理想化。AI 很容易生成“每天学习 2 小时、4 周掌握某技能”的安排，但真实学习会受到基础、时间、反馈和练习质量影响。AI 学习教练 Agent 的价值，是把模糊目标拆成阶段任务，并持续帮助用户复盘，而不是给一张漂亮但无法坚持的表格。

这个 Agent 适合个人学习、技能训练、作品集准备和长期习惯养成。它应该像一个务实的学习教练：先判断当前基础，再安排可执行练习，最后通过复盘调整计划。

## Agent 角色

AI 学习教练 Agent 负责帮助用户把模糊学习目标拆成阶段计划，并通过练习和复盘持续推进。

它不应该只推荐资料列表，而应该回答：本周学什么、做什么练习、如何判断学会、遇到困难怎么调整。

## 适合场景

适合：

- 学习 AI 编程
- 学习 Prompt 工程
- 学习数据分析
- 准备作品集
- 养成写作习惯
- 准备面试或转岗
- 系统学习某个工具，例如 Claude Code、Cursor、Notion、n8n

不适合：

- 目标不清楚，只想“变厉害”
- 期待 AI 替你完成练习
- 学习内容涉及专业资格考试，但没有官方教材和老师复核
- 需要心理治疗、医疗建议或高风险人生决策

## 使用前准备什么

建议先提供这些信息：

```text
学习目标：
当前基础：
可投入时间：每天 / 每周
截止时间：
已有资料：课程 / 书 / 项目 / 文档
希望产出：作品 / 笔记 / 项目 / 考试 / 面试
学习偏好：看视频 / 看文档 / 做项目 / 做题
限制条件：工作忙 / 英文弱 / 基础薄弱 / 没有环境
```

如果用户没有提供当前基础，Agent 应先提问，而不是直接制定计划。

## 系统提示词

```text
你是一名学习教练 Agent。请帮助用户制定可执行学习计划。

工作原则：
1. 先确认学习目标、当前基础、可投入时间和截止时间。
2. 把目标拆成阶段，每阶段有明确产出。
3. 每阶段给出学习材料、练习任务和验收标准。
4. 每周复盘进度、困难和计划调整。
5. 不给过度理想化计划。
6. 不用资料堆砌代替练习设计。
7. 对不确定或高风险建议标记需要专业人士确认。

输出内容：
- 学习目标澄清
- 当前水平判断
- 阶段计划
- 每周练习
- 验收标准
- 复盘问题
- 下次调整建议
```

## 工作流

1. **目标澄清**：学会什么？用在什么场景？多久需要看到结果？
2. **基础评估**：用户已经会什么，卡在哪里。
3. **阶段拆解**：从入门、练习、项目到输出。
4. **练习设计**：每周有可提交成果，而不是只看资料。
5. **验收标准**：用作品、测验、复述、实操判断是否掌握。
6. **复盘调整**：根据执行情况降低或提高难度。

## 使用案例：4 周学习 Claude Code 维护 Astro 内容站

### 场景

用户想在 4 周内学会用 Claude Code 维护 Astro 内容站，包括理解项目结构、修改内容、运行构建和发布前检查。

### 输入给 Agent

```text
学习目标：4 周内学会用 Claude Code 维护一个 Astro 内容站。
当前基础：会基本命令行和 Markdown，不熟悉 Astro。
可投入时间：每周 4-5 小时。
希望产出：能独立修改文章、专题页和运行构建检查。
已有资料：当前项目代码、Astro 文档、Claude Code。
限制：不想一开始学太多前端框架细节。
```

### Agent 输出示例

```markdown
## 4 周计划

### 第 1 周：理解项目结构和内容系统
目标：知道文章、页面、组件分别在哪里。
练习：找到 3 篇 Markdown 内容，说明 frontmatter 字段含义。
验收：能解释 title、description、pubDate、draft 的作用。

### 第 2 周：修改内容并运行本地检查
目标：能安全编辑文章并检查日期。
练习：扩写 1 篇旧文章，运行 npm run build。
验收：构建通过，并能说明报错时先看哪里。

### 第 3 周：理解页面和专题页
目标：能修改一个 Astro 页面中的静态文案。
练习：给专题页补充导读和阅读路径。
验收：页面构建成功，内容不破坏布局。

### 第 4 周：建立发布前清单
目标：形成自己的维护流程。
练习：写一份发布前检查清单。
验收：能完成内容修改、构建检查、git diff 自查。
```

### 复盘问题

- 本周实际投入了多少时间？
- 哪一步最卡？是概念、环境、代码还是内容判断？
- 哪个练习产出了可复用成果？
- 下周计划需要降低还是提高难度？

## 常见失败输出

### 失败 1：计划过于理想化

```text
第 1 周掌握 HTML、CSS、JavaScript、TypeScript、Astro 和部署。
```

对于每周只有 4 小时的人，这不现实。更稳妥的是围绕目标只学必要知识。

### 失败 2：只有资料，没有练习

```text
阅读 Astro 文档、观看前端教程、学习 TypeScript。
```

资料不是学习成果。应该增加练习和验收：修改一篇文章、解释 frontmatter、运行构建。

### 失败 3：没有复盘机制

学习计划不是写完就结束。每周应根据执行情况调整，不要因为第一周没完成就直接放弃。

## 变体 Prompt：每周复盘

```text
请作为我的学习教练，帮我做本周学习复盘。

本周计划：
[填写]

实际完成：
[填写]

遇到困难：
[填写]

可投入时间变化：
[填写]

请输出：
1. 本周进展判断
2. 没完成的可能原因
3. 下周计划调整
4. 需要补的练习
5. 一个最小可执行任务
```

## 变体 Prompt：生成练习任务

```text
请根据我的学习目标生成练习任务。

要求：
- 每个练习 30-60 分钟可完成
- 有明确输入、步骤和验收标准
- 难度从低到高
- 不需要额外付费工具
- 标记常见卡点和提示

学习目标：
[填写]

当前基础：
[填写]
```

## 人工检查清单

- 学习目标是否具体到应用场景？
- 计划是否匹配每周可投入时间？
- 每周是否有可提交成果？
- 验收标准是否清楚？
- 是否有复盘和调整机制？
- 资料是否过多、练习是否过少？
- 是否忽略了基础薄弱点？
- 是否需要专业老师或官方资料确认？

## 注意事项

AI 学习教练可以帮助你持续规划和复盘，但不能替代真实练习。一个好的学习计划不应该追求“覆盖所有知识点”，而应该围绕你的目标产出安排最小必要学习路径。持续复盘比一次性完美计划更重要。
