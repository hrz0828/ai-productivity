SEO 编辑不是把关键词塞进标题和正文。真正有价值的 SEO 编辑要判断：搜索者想解决什么问题，当前文章是否回答了这个问题，内容是否有足够案例和可信度，页面是否能引导读者继续阅读。AI SEO 编辑 Agent 可以帮助内容站把这些检查流程标准化。

这个 Agent 适合内容站作者、SEO 编辑、独立开发者和运营人员使用。它可以辅助关键词拆解、搜索意图分析、SEO Brief、标题描述优化、内容缺口检查、FAQ 和内链建议。但它不应该为了排名牺牲准确性，也不能编造搜索量、案例、评测数据或用户反馈。

## Agent 角色

AI SEO 编辑 Agent 负责帮助内容站规划和优化文章，使内容更贴近搜索需求，同时保持可读性和实用价值。

它应该像一个审稿编辑，而不是关键词工具：先判断读者问题，再评估文章是否真正解决问题。

## 目标

- 分析搜索意图
- 生成文章 Brief
- 优化标题和描述
- 检查内容覆盖度
- 提供内链和 FAQ 建议
- 标记需要真实经验或案例补充的位置
- 识别关键词堆砌、薄内容和过度承诺

## 适合场景

适合：

- 新文章写作前生成 SEO Brief
- 旧文章翻新和内容缺口检查
- Prompt、Workflow、Tool、Guide 页面优化
- 文章标题和描述改写
- 专题页内部链接规划
- Search Console 查询词复盘

不适合：

- 批量生成没有人工编辑的 SEO 文章
- 编造搜索量、CTR、排名或用户数据
- 把关键词密度作为主要优化目标
- 为医疗、法律、金融等高风险主题提供未经复核的建议

## 系统提示词

```text
你是一名 SEO 编辑 Agent，擅长把用户搜索意图转化为高质量文章结构。

原则：
1. 先理解目标关键词背后的真实问题。
2. 文章结构要服务读者，不堆砌关键词。
3. 标题和描述要准确，不夸大。
4. 给出内链建议和 FAQ。
5. 标记需要真实经验、截图、数据或案例补充的位置。
6. 不编造搜索量、排名、工具功能或用户评价。
7. 对低价值内容风险提出具体修改建议。

输出：
- 搜索意图分析
- 内容类型建议
- SEO Brief
- 标题和描述建议
- 内容缺口
- 内链建议
- FAQ
- 人工补充清单
```

## 使用方式

每次输入目标关键词、目标读者、已有草稿或参考资料，让 Agent 输出可执行的优化建议。

输入模板：

```text
目标关键词：
目标读者：
文章类型：教程 / 模板 / 对比 / 清单 / 指南 / 案例
站点定位：
已有草稿：
希望引导的下一步阅读：
不能写的内容：
```

## 工作流

1. **搜索意图分析**：判断读者是想学习概念、找模板、做对比还是解决具体问题。
2. **内容类型选择**：确定文章更适合教程、模板、清单、对比还是案例。
3. **Brief 生成**：列出 H2/H3、每节要回答的问题和需要素材。
4. **草稿审查**：检查是否满足意图、是否空泛、是否缺案例。
5. **标题描述优化**：准确说明页面价值，不夸大。
6. **FAQ 和内链**：补充常见问题和站内阅读路径。
7. **发布后复盘**：根据查询词和点击数据调整内容。

## 使用案例：优化一篇 Prompt 教程

### 场景

你写了一篇“AI 写周报 Prompt”，希望让它更适合搜索引擎和读者阅读。

### 示例任务

```text
请作为 SEO 编辑 Agent，审查这篇文章。

目标关键词：AI 写周报 Prompt
目标读者：需要快速完成周报的职场人士
文章类型：模板 + 教程
请检查标题、描述、大纲、FAQ、内链和内容缺口。

已有草稿：
[粘贴文章]
```

### Agent 输出示例

```markdown
## 搜索意图
读者主要想获得可复制的 Prompt，并理解如何输入自己的工作内容。文章不应只解释周报概念。

## 内容缺口
- 需要增加输入示例和输出示例。
- 需要说明如何避免周报写成流水账。
- 需要增加不同角色的 Prompt 调整方式。
- 需要补充发送前人工检查清单。

## 标题建议
AI 写周报 Prompt：把零散工作事项整理成结构化汇报

## 内链建议
- 链接到 AI 邮件写作 Prompt，承接职场沟通场景。
- 链接到 AI OKR Prompt，帮助读者做目标复盘。
- 链接到会议纪要 Prompt，承接会议记录整理需求。
```

## 常见失败输出

### 失败 1：只建议增加关键词

```text
建议多出现几次“AI 写周报 Prompt”。
```

这类建议价值很低。更好的建议应该是补充搜索者需要的内容：Prompt、输入示例、输出示例、失败案例和检查清单。

### 失败 2：标题过度承诺

```text
用了这个 AI Prompt，周报效率提升 10 倍。
```

如果没有真实测试数据，不要写。更稳妥：

```text
AI 写周报 Prompt：把零散工作事项整理成结构化汇报
```

### 失败 3：生成泛泛 FAQ

```text
问：AI 是什么？
答：AI 是人工智能。
```

FAQ 应服务当前搜索意图，例如：

- 没有具体数据时怎么写周报？
- AI 生成的周报可以直接发吗？
- 如何让周报更像自己的表达？

## 变体 Prompt：旧文翻新审查

```text
请作为 SEO 编辑 Agent，审查下面这篇旧文章是否存在低价值内容风险。

重点检查：
1. 是否满足目标关键词的搜索意图
2. 是否只有模板，没有使用说明
3. 是否缺少案例、失败输出和人工检查清单
4. 标题和描述是否准确
5. 是否需要增加 FAQ 和内链
6. 是否存在过度承诺或编造数据

请按优先级输出修改建议。

目标关键词：
[填写]

文章：
[粘贴]
```

## 变体 Prompt：Search Console 查询词复盘

```text
请根据下面的 Search Console 查询词数据，判断文章是否需要更新。

输入：
- 页面主题：
- 查询词、曝光、点击、CTR：
- 当前标题和描述：

请输出：
1. 查询词反映的读者意图
2. 当前文章可能没有覆盖的问题
3. 标题和描述是否需要调整
4. 建议新增章节或 FAQ
5. 不建议追的无关查询词
```

## 人工检查清单

- 标题是否准确匹配搜索意图？
- 文章开头是否直接回答读者问题？
- 是否有具体步骤、示例或模板？
- 是否有真实经验或人工判断？
- 是否避免关键词堆砌？
- 是否没有编造搜索量、数据或案例？
- SEO 描述是否具体而不夸大？
- FAQ 是否来自真实读者问题？
- 内链是否有阅读路径，而不是随意堆链接？
- 是否记录更新时间并检查过时信息？

## 使用边界

SEO 优化不等于堆关键词。最终文章仍应以读者能否完成任务为第一标准。AI SEO 编辑 Agent 可以帮助你发现内容缺口，但真正的内容质量来自真实案例、清晰步骤、可信来源和持续更新。
