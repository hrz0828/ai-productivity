工作邮件最重要的不是文采，而是让对方清楚知道：发生了什么、你希望对方做什么、什么时候需要回复、如果不处理会有什么影响。AI 很擅长把邮件写得礼貌顺滑，但如果输入目标不清楚，它也可能写出一封“看起来专业、实际没有行动指向”的邮件。

这个 Prompt 适合生成项目同步、催办、合作沟通、会议邀请、问题反馈等邮件初稿。使用时要特别注意语气：催办不能太冲，反馈不能太模糊，合作邮件不能过度承诺。

## 适用场景

当你需要写项目同步、催办、合作沟通、会议邀请或问题反馈邮件时，可以用这个 Prompt 快速生成初稿。

适合：

- 向同事确认进度
- 向负责人同步风险
- 向合作方说明需求
- 发送会议邀请或会后跟进
- 把口语化想法整理成正式邮件

不适合：

- 涉及法律、合同、赔偿、解约等高风险沟通
- 涉及敏感人事、绩效或冲突处理
- 需要代表公司做正式承诺
- 邮件中包含不应上传给 AI 的客户隐私或商业机密

## 使用前先明确 4 件事

1. **邮件目标**：你到底希望对方做什么？确认、回复、审批、补充材料还是采取行动？
2. **收件人关系**：直属领导、平级同事、外部客户、供应商，语气不同。
3. **背景信息**：对方是否已经知道这件事？需要补多少上下文？
4. **截止时间**：如果需要回复，最好明确时间点。

很多邮件写不清楚，不是因为表达能力问题，而是目标没有想清楚。

## Prompt 正文

```text
你是一名专业的职场沟通写作助理。请根据以下信息帮我写一封工作邮件。

请先理解邮件目标，再生成邮件正文。

输入信息：
- 收件人：
- 收件人与我的关系：
- 邮件目标：
- 背景信息：
- 需要对方做什么：
- 截止时间：
- 语气：正式 / 友好 / 简洁 / 委婉
- 不应提及的信息：

输出要求：
1. 邮件标题
2. 邮件正文
3. 如果适合，补充一个更简洁版本
4. 标记语气可能过强或过弱的地方
5. 不要夸大，不要加入未提供的承诺
```

## 输入示例

```text
收件人：研发负责人
收件人与我的关系：跨部门协作同事
邮件目标：确认搜索功能上线时间
背景信息：页面已经开发完成，等待最后构建验证
需要对方做什么：确认是否可以本周五上线
截止时间：今天下班前回复
语气：简洁、专业
不应提及的信息：不要说“延期会影响绩效”
```

## 输出示例

```text
标题：确认搜索功能本周五上线安排

你好，

搜索功能页面目前已经完成开发，并已进入最后构建验证阶段。想和你确认一下，该功能是否可以按计划在本周五上线。

如果上线时间有调整，麻烦今天下班前同步一下原因和新的预计时间，方便我更新项目计划。

谢谢。
```

## 常见失败输出

### 失败 1：邮件很礼貌，但没有行动要求

错误示例：

```text
希望你了解目前搜索功能的进展，如有问题欢迎沟通。
```

这句话太弱，对方不知道是否需要回复。可以改成：

```text
麻烦你今天下班前确认是否可以按本周五上线；如果不能，请同步原因和新的预计时间。
```

### 失败 2：催办语气太冲

错误示例：

```text
你们一直没有回复，已经影响了项目进度，请尽快处理。
```

如果不是正式升级问题，这样写容易引发对立。更稳妥：

```text
想跟进一下这个事项的当前进展。由于我需要在今天更新项目计划，麻烦你下班前同步一下是否可以按原计划推进。
```

### 失败 3：AI 加入未提供承诺

AI 可能会写：

```text
我们会确保上线后没有任何问题。
```

这类承诺风险很高。可以在 Prompt 里明确：

```text
不要加入“保证、确保、一定”等承诺性表述，除非我明确提供。
```

## 三种常用邮件变体

### 1. 催办邮件

```text
请帮我写一封委婉但明确的催办邮件。

背景：
[说明事项]

我需要对方：
[具体动作]

截止时间：
[时间]

要求：
- 语气礼貌，不指责
- 明确说明为什么需要这个时间点前回复
- 给出如果无法完成时的替代动作
```

### 2. 风险同步邮件

```text
请帮我写一封项目风险同步邮件。

风险：
[说明风险]

可能影响：
[说明影响]

已采取动作：
[说明动作]

需要支持：
[说明需要谁做什么]

要求：
- 不夸大风险
- 不回避问题
- 让收件人知道需要做什么决策
```

### 3. 合作沟通邮件

```text
请帮我写一封外部合作沟通邮件。

合作背景：
[背景]

希望对方提供：
[材料或动作]

我方可以提供：
[我方信息]

边界：
[不能承诺的事项]

要求：
- 语气专业友好
- 不做超出授权的承诺
- 信息结构清晰
```

## 调试建议

如果 AI 写得太正式，可以追加：

```text
请改得更像真实同事之间的邮件，保留礼貌，但不要太像公告。
```

如果 AI 写得太软，可以追加：

```text
请让行动要求更明确，但不要使用指责语气。
```

如果邮件太长，可以追加：

```text
请压缩到 150 字以内，只保留背景、请求和截止时间。
```

## 发送前检查清单

- 收件人是否正确？
- 邮件目标是否一句话能说清？
- 是否明确希望对方做什么？
- 是否有截止时间？
- 语气是否符合关系和场景？
- 是否包含未经授权的承诺？
- 是否泄露敏感信息？
- 是否需要抄送相关负责人？

## 使用建议

邮件 Prompt 最重要的是明确“希望对方做什么”。如果只提供背景，AI 可能写成说明文，而不是行动导向的邮件。生成后建议自己读一遍：如果你是收件人，能不能立刻知道下一步该做什么？如果不能，就继续改。
