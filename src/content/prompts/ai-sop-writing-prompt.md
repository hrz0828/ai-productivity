## 适用场景

当团队里有一项工作需要反复执行，但每个人做法不一致、交接成本高、容易遗漏步骤时，可以使用这个 Prompt 生成 SOP（标准作业流程）。

适合整理的流程包括：

- 内容发布流程
- 客服问题处理流程
- 活动上线流程
- 数据日报流程
- 需求评审流程
- 新人入职流程
- 线上故障处理流程

## Prompt 正文

```text
你是一名流程管理顾问，擅长把重复工作整理成清晰、可执行的 SOP（标准作业流程）。

请根据我提供的工作信息，生成一份 SOP 初稿。

请按以下结构输出：
1. SOP 名称
2. 适用场景
3. 适用对象
4. 流程目标
5. 输入材料
6. 输出结果
7. 角色和职责
8. 标准执行步骤
9. 每一步的检查点
10. 常见异常和处理方式
11. 质量标准
12. 复盘和更新机制

写作要求：
- 步骤必须具体到可执行动作
- 每一步说明负责人、输入、动作和输出
- 不要只写原则，要写检查点
- 标记需要人工确认或审批的环节
- 如果信息不足，请列出待补充问题

工作信息：
流程名称：
适用团队：
触发条件：
当前做法：
常见问题：
参与角色：
需要产出：
风险或限制：
```

## 输入示例

```text
流程名称：SEO 文章发布流程
适用团队：内容运营团队
触发条件：每周发布新文章
当前做法：选题后写文章，编辑检查后发布
常见问题：标题不统一、内链容易遗漏、发布后没人提交索引
参与角色：作者、编辑、站点维护者
需要产出：已发布文章、SEO 标题、描述、内链、Search Console 提交记录
风险或限制：不能发布未经核实的数据，不能使用侵权图片
```

## 输出示例

```markdown
# SEO 文章发布 SOP

## 1. 适用场景
适用于内容运营团队每周发布 SEO 文章的流程。

## 2. 流程目标
确保每篇文章在发布前完成标题、摘要、内链、事实核查和索引提交，减少低质量内容和遗漏操作。

## 3. 角色和职责
| 角色 | 负责事项 |
| --- | --- |
| 作者 | 选题、写作、补充资料来源 |
| 编辑 | 检查结构、事实、标题和内链 |
| 站点维护者 | 发布文章、检查页面、提交索引 |

## 4. 标准执行步骤

### Step 1：确认选题
- 负责人：作者
- 输入：关键词、用户问题、已有内容列表
- 动作：确认文章要解决的问题和目标读者
- 输出：选题说明
- 检查点：是否与已有文章重复？是否有明确搜索意图？

### Step 2：撰写初稿
- 负责人：作者
- 输入：选题说明、参考资料
- 动作：完成正文、示例和检查清单
- 输出：文章初稿
- 检查点：是否有实际操作价值？是否有未经验证的数据？

### Step 3：编辑检查
- 负责人：编辑
- 输入：文章初稿
- 动作：检查标题、摘要、结构、内链和术语解释
- 输出：可发布版本
- 检查点：是否至少包含 2 个相关内链？是否有 SEO description？

### Step 4：发布和索引提交
- 负责人：站点维护者
- 输入：可发布版本
- 动作：发布文章、检查页面、提交 Search Console
- 输出：线上 URL 和提交记录
- 检查点：页面是否可访问？移动端是否正常？

## 5. 常见异常
- 如果缺少资料来源：退回作者补充。
- 如果主题与已有文章高度重复：合并或改为补充段落。
- 如果页面发布后显示异常：暂停传播并修复页面。
```

## 使用技巧

SOP 的重点不是“写得完整”，而是“让不同人按同一套流程执行”。因此要让 AI 输出负责人、输入、动作、输出和检查点，而不是只输出大段说明。

## 可调整参数

- 流程粒度：简版清单、详细 SOP、培训版 SOP
- 适用对象：新人、执行同学、管理者、跨部门团队
- 输出格式：Markdown、表格、检查清单
- 风险重点：质量、合规、安全、效率、协作

## 人工复核重点

- 每一步是否真的可执行？
- 是否明确负责人？
- 是否有输入和输出？
- 是否有质量检查点？
- 是否说明异常情况怎么处理？
- 是否需要审批或记录留痕？
