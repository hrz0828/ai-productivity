## 适用场景

当你已经有一个功能想法，但还没有完整 PRD 结构时，可以使用这个 Prompt 快速生成初稿。

## Prompt 正文

```text
你是一名资深产品经理。请根据我提供的功能想法，生成一份产品需求文档初稿。

请包含：
1. 需求背景
2. 目标用户
3. 用户问题
4. 产品目标
5. 核心流程
6. 功能需求
7. 非功能需求
8. 数据指标
9. 风险与依赖
10. 待确认问题

要求：
- 明确区分事实、假设和待确认问题
- 不要编造业务数据
- 功能描述要可被设计和研发理解
- 对不完整的信息提出澄清问题

功能想法：
[粘贴内容]
```

## 使用技巧

建议先输入业务目标和用户场景，再让 AI 生成 PRD。如果只输入功能名称，AI 容易生成过于泛化的内容。

如果需求还不清楚，先不要直接写 PRD。可以先使用 [用 AI 做需求分析的完整工作流](/workflows/demand-analysis-with-ai/) 判断需求是否真实、是否值得做，再进入 PRD 写作。也可以从 [AI 产品经理专题](/topics/ai-product-manager/) 按推荐顺序阅读。

## 使用案例：为内容站增加搜索功能

### 场景

你运营一个 AI 内容站，希望增加站内搜索，用户可以搜索 Prompt、Workflow 和工具教程。

### 示例输入

```text
功能想法：增加站内搜索页面，用户输入关键词后可以搜索所有文章。
目标用户：访问内容站的普通读者。
目标：提高内容发现效率。
约束：第一版不做登录，不记录用户搜索历史。
```

### 期望输出

AI 应生成 PRD 初稿，包括搜索入口、搜索结果展示、无结果状态、移动端适配、数据指标和不做范围。

### 人工复核重点

PRD 需要明确技术方案是否依赖第三方服务、搜索结果是否需要高亮、是否支持中文分词。这些问题不能只停留在“支持搜索”四个字。

## 输出示例

```markdown
# 站内搜索功能 PRD 初稿

## 1. 背景
随着内容数量增加，用户仅通过栏目浏览查找 Prompt 和 Workflow 的效率下降，需要提供站内搜索能力。

## 2. 产品目标
帮助用户通过关键词快速找到相关文章，提升内容发现效率和页面浏览深度。

## 3. 目标用户
- 首次访问本站、带着明确问题搜索的用户
- 想快速找到某个 Prompt 模板的老用户

## 4. 功能需求
### 搜索入口
首页 Hero 区和 Header 提供搜索入口。

### 搜索页面
用户输入关键词后展示匹配文章标题、摘要和链接。

### 无结果状态
当没有结果时，提示用户更换关键词，并提供热门栏目入口。

## 5. 不做范围
- 不做用户搜索历史
- 不做个性化推荐
- 不做登录态收藏

## 6. 待确认问题
- 是否需要中文分词优化？
- 是否统计搜索关键词？
```
