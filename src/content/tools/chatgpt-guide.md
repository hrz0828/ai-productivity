ChatGPT 是很多人接触 AI 工具的入口，但“会聊天”和“能稳定完成工作任务”是两回事。日常问答可以随便问，工作场景则需要提供目标、背景、材料、格式和约束。否则 ChatGPT 很容易输出一段看似合理、实际不可用的泛泛内容。

这篇教程介绍 ChatGPT 在写作、总结、研究、头脑风暴和工作流设计中的实用方法，并给出可复制 Prompt、常见失败案例和人工复核清单。

## 工具简介

ChatGPT 是通用 AI 对话工具，适合日常问答、文档写作、内容创作、资料总结和任务拆解。

适合：

- 写邮件、周报、总结
- 生成文章大纲
- 梳理学习计划
- 改写文案
- 头脑风暴
- 设计 Prompt 和 Workflow
- 整理会议记录和行动项
- 生成表格、清单和初稿

不适合：

- 不提供事实材料就让它生成真实报告
- 直接确认法律、医疗、金融等专业结论
- 编造引用、数据、案例或用户反馈
- 把输出不复核就对外发布
- 上传不应外传的客户隐私或公司机密

## 使用方法

不要只问“帮我写”。更好的方式是提供角色、目标、背景、输入材料、输出格式和约束。

一个通用结构：

```text
你是：[角色]
我要完成：[任务目标]
背景是：[上下文]
输入材料：[粘贴材料]
输出格式：[表格 / Markdown / 邮件 / 大纲 / JSON]
要求：[风格、长度、限制]
不要：[不能编造、不能承诺、不能提及]
```

## 新手使用步骤

### Step 1：把任务说清楚

先说明你要完成什么，而不是只给一个模糊指令。

```text
我需要写一封发给合作方的邮件，目的是确认上线时间和对方需要补充的材料。
```

不推荐：

```text
帮我写邮件。
```

### Step 2：补充上下文

ChatGPT 的输出质量很依赖上下文。建议补充：

- 目标读者是谁
- 当前背景是什么
- 已有材料有哪些
- 希望语气正式还是轻松
- 输出要多长
- 哪些内容不能编造
- 是否需要行动项或下一步

### Step 3：指定输出格式

如果你要复用结果，最好明确格式：Markdown、表格、邮件正文、JSON、清单或大纲。

例如：

```text
请用表格输出，字段包括：任务、负责人、截止时间、风险、待确认问题。
```

### Step 4：让 ChatGPT 先给结构

复杂任务不要直接生成全文，可以先让它给大纲或问题清单。

```text
请先不要写正文。请先给我一个大纲，并指出还需要我补充哪些信息。
```

这种方式适合 PRD、研究报告、SEO 文章、项目复盘和 PPT。

### Step 5：用反馈迭代

不要只说“再好一点”。更有效的反馈是指出具体问题：太空泛、语气太营销、缺少例子、没有行动项、格式不对。

可用反馈：

```text
这版太像宣传文案。请改成更克制的教程语气，并增加一个真实使用示例。
```

```text
请保留结构，但把每个要点改成可执行动作。
```

## 示例 Prompt：写文章大纲

```text
你是一名内容编辑。请根据以下主题生成一篇文章大纲。

目标读者：AI 工具新手
文章目标：帮助读者理解如何使用 ChatGPT 提升办公效率
要求：
- 结构清晰
- 包含示例和注意事项
- 不夸大效率提升
- 标记需要补充真实案例的位置

输出格式：Markdown 大纲
```

## 示例 Prompt：整理会议行动项

```text
你是一名会议纪要助理。请根据下面的会议记录整理行动项。

要求：
- 先输出会议结论
- 再输出行动项表格
- 字段包括：事项、负责人、截止时间、依赖、待确认问题
- 如果负责人或时间不明确，写“待确认”
- 不要编造会议中没有的信息

会议记录：
[粘贴]
```

## 示例 Prompt：研究资料整理

```text
你是一名研究助理。请整理下面的资料。

要求：
- 区分事实、观点、推测和待验证信息
- 保留来源和日期
- 提炼关键结论
- 标记证据不足的位置
- 给出下一步需要查证的问题

资料：
[粘贴]
```

## 示例 Prompt：设计工作流

```text
请帮我把下面的任务设计成一个可重复使用的工作流。

任务：
[例如：把一篇文章改成短视频脚本]

要求输出：
1. 适用场景
2. 输入材料
3. 分步骤流程
4. 每一步 Prompt
5. 常见失败情况
6. 人工检查清单
```

## 常见误区

### 只给一句话

例如“帮我写周报”。这种输入会让 ChatGPT 自行脑补背景，结果通常很泛。

更好：

```text
请根据下面的本周工作事项，帮我整理成周报。结构包括：本周完成、关键结果、遇到的问题、下周计划。不要编造数据。
```

### 不提供事实材料

如果你没有提供数据、进展和问题，ChatGPT 可能生成看似合理但不真实的内容。涉及简历、汇报、研究报告时尤其要注意。

### 一次要求太多

比如同时要求写文章、做图表、给标题、生成推广文案。复杂任务建议拆成多步：先大纲，再正文，再标题，再检查。

### 不做人工复核

ChatGPT 很适合初稿和结构整理，但不适合直接替你确认事实、数据和决策。

### 反馈太模糊

“再专业一点”很难执行。可以改成：

```text
请减少形容词，增加具体步骤，并把每段控制在 100 字以内。
```

## 常见失败输出和修正

### 失败 1：空泛开头

```text
在当今快速发展的时代，AI 正在改变我们的工作方式。
```

修正 Prompt：

```text
请直接从读者的具体问题开始，不要使用“在当今时代”这类泛化开头。
```

### 失败 2：编造数字

```text
使用这个方法可以让效率提升 300%。
```

修正 Prompt：

```text
不要写任何我没有提供的数据。如果需要数据，请用 [需要补充数据] 标记。
```

### 失败 3：行动项不明确

```text
后续需要继续优化和加强沟通。
```

修正 Prompt：

```text
请把行动项改成具体动作，包含负责人、截止时间和验收标准。未知信息写“待确认”。
```

## 隐私和安全注意事项

使用 ChatGPT 前，先判断材料是否可以上传：

- 客户姓名、电话、邮箱、合同信息要脱敏
- 公司内部策略、账号、密钥、财务数据不要直接粘贴
- 面试、简历和绩效内容要删除无关个人信息
- 对外发布前检查是否包含内部链接或敏感截图
- 涉及医疗、法律、投资等专业问题，需要专业人士复核

## 人工复核清单

- 是否有编造的数据或事实？
- 输出是否符合指定格式？
- 是否遗漏关键背景？
- 语气是否适合目标读者？
- 是否有可执行的下一步？
- 是否需要删除空泛表达？
- 是否包含敏感信息？
- 是否需要来源或证据？
- 是否把假设写成了结论？

## 进阶建议：把任务变成工作流

当你发现某类任务经常重复，例如写周报、整理会议纪要、做 SEO Brief，不要每次重新问。可以把稳定步骤沉淀成工作流：

1. 输入模板
2. 第一步理解和澄清
3. 第二步生成结构
4. 第三步生成正文或表格
5. 第四步检查失败点
6. 第五步人工复核

这样 ChatGPT 就不只是聊天工具，而是你的工作流程助手。
