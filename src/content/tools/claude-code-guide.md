## 工具简介

Claude Code 是 Anthropic 提供的 AI 编程助手，可以在终端中读取项目、编辑文件、运行命令，并帮助开发者完成代码理解、功能实现、Bug 修复和重构任务。

## 适合谁使用

- 独立开发者
- 前端和后端工程师
- 产品型创业者
- 需要维护代码项目的非专业开发者

## 能解决什么问题

Claude Code 适合处理：

- 快速理解陌生代码库
- 实现中小型功能
- 修复明确 Bug
- 编写和运行测试
- 生成技术文档
- 批量重构和迁移

## 常见使用场景

### 代码库探索

你可以让 Claude Code 找到路由、数据模型、组件结构或某个功能的实现位置。

### 功能开发

对于多文件功能，先让 Claude Code 制定计划，再确认实现步骤，可以减少返工。

### 代码审查

让 Claude Code 从正确性、安全性、性能和可维护性角度审查改动。

## 推荐使用流程

### Step 1：先让它理解项目

进入陌生项目时，不要一开始就要求修改代码。可以先问：

```text
请帮我快速理解这个项目的目录结构、主要页面、内容来源和构建方式。先不要修改文件。
```

### Step 2：让它制定计划

多文件任务建议先进入计划模式：

```text
我想优化首页专题入口。请先阅读相关文件，给出实现计划、会影响哪些文件，以及需要验证什么。先不要写代码。
```

### Step 3：小步修改

让 Claude Code 分步骤修改，每一步都说明改了什么。避免一次性大改多个模块。

### Step 4：运行验证

修改后至少运行构建或测试：

```text
请运行项目构建，检查是否有类型错误或路由错误，并根据结果修复。
```

### Step 5：总结变更

最后让它总结改动文件、验证结果和后续建议，便于你做代码审查或提交。

## 常见误区

- 没有给目标，只说“优化一下”。
- 没让它先读项目结构，直接要求写代码。
- 一次性要求修改太多文件。
- 没有运行测试或构建。
- 对删除文件、发布、部署等操作没有确认。

## 人工复核清单

- 是否理解了需求和边界？
- 是否改动了不相关文件？
- 是否遵循项目现有风格？
- 是否运行了必要测试或构建？
- 是否有未说明的副作用？
- 是否涉及外部发布、删除、部署等高风险动作？

## 注意事项

对数据库迁移、线上部署、删除文件、发布内容等不可逆操作，需要先确认计划并做好备份。

## 使用案例：用 Claude Code 修改内容站首页

### 场景

你发现首页按钮颜色不清晰，希望让 Claude Code 找到对应文件并修改样式。

### 示例指令

```text
首页“查看工作流”按钮对比度不够，请找到对应代码并把按钮改成更清晰的蓝色主按钮。修改后说明改了哪个文件。
```

### Claude Code 会做什么

它会搜索首页文件，定位按钮 class，修改 Tailwind 样式，并在需要时运行构建确认没有破坏页面。

### 使用建议

对于这类小改动，可以直接让 Claude Code 执行；对于多文件功能，建议先要求它制定计划，再确认实现。
