## 这个 Skill 解决什么问题

团队里很多工作依赖个人经验，交接时容易遗漏。SOP 生成 Skill 可以把经验转成标准流程，降低重复沟通成本。

## 输入要求

- 工作目标
- 适用范围
- 当前操作步骤
- 常见异常
- 质量标准
- 负责人角色

## 执行步骤

1. 提炼流程目标。
2. 识别角色和前置条件。
3. 整理步骤顺序。
4. 添加检查点。
5. 补充异常处理和交付标准。

## 输出格式

```markdown
## SOP 名称
## 适用范围
## 角色分工
## 前置条件
## 操作步骤
## 检查清单
## 异常处理
## 交付标准
```

## 使用建议

生成后最好让实际执行者试跑一次，根据试跑结果补充缺失步骤。

## 使用案例：内容发布 SOP

### 场景

AI 内容站需要持续发布文章，但每次发布都要检查 Frontmatter、内链、SEO 描述和构建结果。可以用 SOP 固化流程。

### 输入示例

```text
目标：发布一篇新的 Workflow 文章
步骤：选题、写作、检查标题描述、添加内链、运行构建、提交 GitHub
质量标准：页面可访问，SEO 字段完整，无构建错误
```

### 输出示例

SOP 应包含适用范围、角色分工、前置条件、操作步骤、检查清单和异常处理。

### 复用方式

把 SOP 放进团队文档或仓库 README，后续每次发布内容都按清单执行，减少遗漏。
