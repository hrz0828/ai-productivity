## 这个 Skill 解决什么问题

SEO 文章质量不稳定，往往是因为写作前没有清晰简报。SEO Brief 生成 Skill 用于在写作前明确目标关键词、搜索意图、读者问题、文章结构和内链策略，让作者不是从空白页开始写。

它适合这些场景：

- 为新文章准备写作大纲和关键词方向。
- 把一个宽泛主题拆成更具体的长尾关键词。
- 在写作前判断用户真正想解决什么问题。
- 给外包作者或团队成员提供统一写作要求。
- 更新旧文章前，先整理缺失小节和 FAQ。

## 输入要求

- 目标关键词：主关键词和可选长尾词。
- 目标读者：新手、进阶用户、开发者、运营、产品经理等。
- 业务目标：流量、转化、注册、阅读引导或品牌认知。
- 参考文章：竞品文章、已有站内文章或搜索结果摘要。
- 必须覆盖的问题：读者一定会问的问题。
- 站内可链接内容：相关 Prompt、Skill、Workflow、Tool 或 Guide。
- 内容边界：不写什么，避免文章发散。

## 使用方法

### 第一步：确认搜索意图

先让 AI 判断关键词背后的意图，是学习概念、找教程、比较工具，还是解决具体问题。

```text
请分析“Claude Code 入门”的搜索意图，并判断读者最可能需要哪些信息。
```

### 第二步：拆分读者问题

一个好的 Brief 不只是大纲，还应该列出读者进入页面后希望得到回答的问题。

```text
请列出这个关键词下读者最关心的 10 个问题，并按重要程度排序。
```

### 第三步：生成文章结构

让 AI 输出 H2/H3 层级结构，并说明每一节要回答什么问题。

### 第四步：补充 FAQ 和内链

FAQ 适合覆盖长尾问题，内链适合把读者引导到更具体的模板和工作流。

## 推荐 Prompt 模板

```text
你是一个 SEO Brief 生成 Skill。请为以下文章生成写作简报。

目标关键词：
目标读者：
业务目标：
参考文章或搜索结果：
必须覆盖的问题：
站内可链接内容：
内容边界：

请输出：
1. 搜索意图
2. 读者核心问题
3. 推荐标题和 SEO description
4. H2/H3 文章大纲
5. 每节写作要点
6. FAQ
7. 内链建议
8. 需要补充的实操素材
```

## 输出格式

```markdown
## 目标关键词
## 搜索意图
## 读者问题
## 推荐标题
## SEO 描述
## 文章大纲
## FAQ
## 内链建议
## 素材清单
```

## 使用案例一：为“Claude Code 入门”生成写作简报

### 场景

你准备写一篇 Claude Code 入门教程，希望先明确读者搜索意图和文章结构。

### 输入示例

```text
目标关键词：Claude Code 入门
目标读者：第一次使用 Claude Code 的开发者
业务目标：获得搜索流量，并引导阅读 AI 编程工作流内容
站内可链接内容：AI 编程助手使用指南、代码审查 Skill、Claude Code 工具教程
```

### 预期输出方向

SEO Brief 应包括搜索意图、读者问题、推荐标题、文章结构、FAQ、内链建议和需要补充的实操截图。

## 使用案例二：更新旧文章 Brief

### 场景

一篇旧文章已有流量，但停留时间低。你希望用 Skill 找出内容缺口。

### 输入示例

```text
文章主题：AI 写周报 Prompt
现有内容：包含模板和简单示例
问题：用户看完后可能不知道如何适配自己的岗位
目标：补充岗位案例、常见错误和改写步骤
```

### 使用建议

让 AI 输出“保留内容、需要新增内容、需要删除内容、内链机会”。更新旧文时，不要只堆新关键词，要补足用户任务链路。

## 质量检查清单

- Brief 是否明确了搜索意图？
- 大纲是否围绕读者问题，而不是围绕作者想写什么？
- 是否有可执行的内链建议？
- FAQ 是否覆盖真实长尾问题？
- 是否要求补充截图、案例或操作步骤？
- 标题是否兼顾关键词和点击动机？

## 注意事项

Brief 不是最终文章。写作时仍然需要真实经验、案例和清晰表达，避免只堆关键词。涉及工具价格、政策、功能状态时，需要在发布前人工核对最新信息。
