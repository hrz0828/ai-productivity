## 这个 Skill 解决什么问题

PRD 写作 Skill 用于把零散需求信息整理成结构化产品需求文档初稿，减少产品经理在文档框架、表达组织和遗漏检查上的重复劳动。

## 输入要求

使用前建议准备以下信息：

- 业务背景
- 目标用户
- 用户痛点
- 产品目标
- 核心功能
- 约束条件
- 期望上线时间
- 已知风险

## 执行步骤

1. 识别需求背景和业务目标。
2. 提炼目标用户和核心场景。
3. 拆分功能模块和用户流程。
4. 生成 PRD 结构。
5. 标记信息不足和需要确认的问题。

## 输出格式

```markdown
# 产品需求文档

## 1. 背景
## 2. 目标
## 3. 用户与场景
## 4. 功能需求
## 5. 非功能需求
## 6. 数据指标
## 7. 风险与依赖
## 8. 待确认问题
```

## 示例输入

```text
我们要为内容站增加一个 Prompt 收藏功能，用户可以收藏常用模板，后续快速访问。
```

## 注意事项

AI 生成的 PRD 是初稿，不应直接替代需求评审。对于边界条件、数据口径、权限规则和异常状态，需要产品经理继续补充。
