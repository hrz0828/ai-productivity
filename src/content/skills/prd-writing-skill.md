## 这个 Skill 解决什么问题

PRD 写作 Skill 用于把零散需求信息整理成结构化产品需求文档初稿，减少产品经理在文档框架、表达组织和遗漏检查上的重复劳动。

## 输入要求

使用前建议准备以下信息：

- 业务背景
- 目标用户
- 用户痛点
- 产品目标
- 核心功能
- 约束条件
- 期望上线时间
- 已知风险

## 执行步骤

1. 识别需求背景和业务目标。
2. 提炼目标用户和核心场景。
3. 拆分功能模块和用户流程。
4. 生成 PRD 结构。
5. 标记信息不足和需要确认的问题。

## 输出格式

```markdown
# 产品需求文档

## 1. 背景
## 2. 目标
## 3. 用户与场景
## 4. 功能需求
## 5. 非功能需求
## 6. 数据指标
## 7. 风险与依赖
## 8. 待确认问题
```

## 示例输入

```text
我们要为内容站增加一个 Prompt 收藏功能，用户可以收藏常用模板，后续快速访问。
```

## 注意事项

AI 生成的 PRD 是初稿，不应直接替代需求评审。对于边界条件、数据口径、权限规则和异常状态，需要产品经理继续补充。

## 使用案例：生成 Prompt 收藏功能 PRD

### 场景

内容站已有 30 多篇 Prompt 和 Workflow，用户希望保存常用内容。你需要快速写出收藏功能 PRD 初稿。

### 输入示例

```text
业务背景：用户需要反复访问常用 Prompt。
目标用户：经常使用本站模板的读者。
核心功能：收藏文章、查看收藏列表、取消收藏。
约束：第一版不做复杂分组，不做公开分享。
```

### Skill 输出

AI 应输出背景、目标、用户故事、功能流程、页面说明、数据指标和待确认问题。对于登录依赖、数据存储和空状态，也应明确标记。

### 评审前检查

- 是否说明收藏入口在哪里？
- 是否说明未登录用户如何处理？
- 是否有取消收藏流程？
- 是否定义成功指标？
